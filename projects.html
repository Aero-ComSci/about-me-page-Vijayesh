9 lives This is a basic hangman game without the hangman gui. The player is given 5 blank spaces and they can either guess a letter or guess the word entirely. If the player gets it right, the clue updates itself to add the letter, but the user loses a life if they get it incorrectly. The game ends when the player guesses the word correctly, or when they use all 9 of their lives. This game would work much better with an actual hangman gui, but that's hard to code in python so I scrapped that. Another feature I could've added was a list of guessed letters.
Clock This is a simple python code that tells the time using turtle as the GUI and datetime to get the actual time. Not much to say about this code. I guess I could've made it better by adding the high school bell schedule to it (wonder where I got that idea from).
Fast Food Order This is an application that allows the user to order items at a fast food place. They have the option of choosing their burger, the size and type of drinks if they want it, the size of french fries if they want it, and how many ketchup packets. At the end, the application prints out the total cost of the order. This application is not case sensitive, so this means that capitalization doesn't matter due to the code changing every input to lowercase. The user can also change their order at the end of the application if they want to, but they have to start all the way from the beginning.
Coin Collector This is a basic game that allows the user to move a character on the screen using the arrow keys in order to catch a coin. After the coin is collected, the coin moves to a random place on the screen, and this continues for 1 minute. After that, the final score is provided. The only major thing to note about this game is that it incorporates pgzrun for the gui, whic is an extension of pygame. This means that the user should have downloaded the pip file in order to use pgzrun.
Shoot the Fruit This is a basic game that allows the user to click on the apple using the cursor. After that, the apple moves to a random place on the screen. Egg Catcher This is a game that allows the user to use the catcher to catch eggs falling from the screen using the arrow keys left and right. As the game progresses, the more and more eggs appear quickly on the screen. This means that the difficulty has increased. The game ends when the user drops three of the eggs. Also nothing much to say here. The eggs have different colors I guess.
Pokemon Game This is basically just a copycat Pokemon game made in python. The user picks a pokemon from a list of three, then they battle the computer's chosen pokemon. The user and computer each get to choose from four moves to use, and the game ends when one pokemon's hp becomes zero. Each pokemon has its own unique attack, defense, and hp statistic. Ok, so there are a lot things that can be improved in this code (that my friends and I are working on for fun). We need to make a better damage equation that actually incorporates both the attack and defense, not just attack, let the moves have their own type and damage output instead of using the pokemon's type for the move, and add an actual gui to this game. One good feature I have is the delay print, which makes everything print slowly just like the original Pokemon games.
